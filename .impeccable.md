## Design Context

### Users
**Developer hobbyists** using ifcli as a personal productivity tool for AI-assisted coding, experimentation, and learning. They use it in personal development environments—terminals and IDEs—often during late-night coding sessions or weekend projects. Primary jobs-to-be-done: quick AI queries without leaving terminal, context-aware assistance for coding tasks, managing multiple chat sessions across projects. The interface needs to feel native to their existing developer workflow and integrate seamlessly with their terminal aesthetic.

### Brand Personality
**Confident. Technical. Focused.**

ifcli speaks the language of developers who know what they want. It's opinionated but not pushy, technical but not intimidating. The brand embodies:
- **Confidence**: Clear visual hierarchy, decisive color choices, reliable feedback. Users trust it to work.
- **Technical credibility**: Uses developer-favored color schemes (Tokyo Night, Catppuccin, Rose Pine), clean typography, minimal visual noise.
- **Focus**: Gets out of the way. Essential information only. No decorative flourishes that distract from the task at hand.

Emotional goals: Users should feel **in control** and **efficient**. The interface should inspire confidence through clarity, not delight through surprise. Think "reliable power tool" not "friendly assistant."

### Aesthetic Direction

**Visual tone**: Terminal-native, dark-mode-first, developer-grade. The interface feels like it belongs in a sophisticated development environment, not a consumer app.

**Design references**:
- Tokyo Night theme family (default) — sets the visual baseline
- Catppuccin and Rose Pine palettes — proven developer-favored color systems
- Modern terminal emulators (Alacritty, Kitty, iTerm2) — clean, high-contrast, customizable

**Anti-references**:
- Consumer chat apps (bright, bubbly, emoji-heavy)
- Enterprise software (cluttered, bureaucratic, patronizing)
- Generic "AI assistant" branding gradients and anthropomorphism

**Theme philosophy**: Dark mode is primary and native. Light themes (Catppuccin Latte, Rose Pine Dawn, Tokyo Night Day) exist for users who prefer them, but they're secondary. The default experience is dark, terminal-authentic, and tuned for long coding sessions.

### Design Principles

1. **Terminal-First Aesthetic**  
   Every visual choice respects terminal constraints and conventions. Use monospace fonts, ASCII/Unicode box drawing characters, and standard terminal colors. Avoid anything that wouldn't render correctly in a modern terminal emulator.

2. **Clarity Over Decoration**  
   Every pixel serves a purpose. The interface uses color and spacing to establish hierarchy, not decoration. Unicode borders and star-flower spinner glyphs add personality without sacrificing readability.

3. **Consistent Visual Language**  
   Three content types (reasoner, tools, assistant) each have dedicated color schemes defined in the theme system. Users recognize content type by color immediately. The visual language is consistent across all themes.

4. **Context-Aware Feedback**  
   The interface provides clear state feedback through color-coded messages (waiting, rendering, error) and the Ora spinner. Users always know what's happening without verbose explanations.

5. **Speed Perception**  
   Real-time rendering (streaming tokens) creates perceived speed even during longer AI responses. The interface prioritizes showing progress over waiting for complete results.

### Technical Constraints

- **Platform**: Terminal CLI via Bun.js runtime
- **Colors**: 16-color terminal palette mapped to theme colors via Chalk
- **Typography**: Monospace, terminal default
- **Spacing**: 4-space indentation, consistent newline patterns for section separation
- **Borders**: Unicode box-drawing characters (rounded, sharp, double, single, or none)
- **Theming**: Theme selection persists in app settings; all UI components reference the active theme

### Implementation Notes

- Theme system lives in `src/component/theme/` — each theme defines terminal colors and chat box colors
- All display components receive `ChalkTerminalColor` and `ChalkChatBoxTheme` from app context
- `SimplifiedDisplay` and `OraShow` handle real-time output and spinner animations
- Color is the primary visual differentiator; ensure themes maintain sufficient contrast
- No accessibility accommodations currently implemented beyond color theming — considered acceptable for personal developer-tool context
